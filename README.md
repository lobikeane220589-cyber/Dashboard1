
# Почему сайт не открывается и как это исправить

Браузеры не понимают файлы `.tsx`. Чтобы сайт заработал на GitHub Pages, проект нужно "собрать".

### Шаг 1: Подготовка
1. Установите на компьютер [Node.js](https://nodejs.org/).
2. Скачайте все файлы вашего репозитория `Dashboard1` в папку на компьютере.

### Шаг 2: Сборка
1. Откройте терминал (или командную строку) в папке с проектом.
2. Введите команду для установки библиотек:
   ```bash
   npm install
   ```
3. Введите команду для сборки проекта:
   ```bash
   npm run build
   ```
4. После этого в папке проекта появится новая папка **`dist`**.

### Шаг 3: Загрузка на GitHub
1. Откройте папку **`dist`**.
2. Загрузите **всё её содержимое** (файлы `index.html`, папки `assets` и т.д.) в ваш репозиторий GitHub `Dashboard1`. 
   *Важно: загружайте именно файлы ИЗ папки dist, а не саму папку dist.*

### Шаг 4: Настройка GitHub Pages
1. В репозитории зайдите в **Settings** -> **Pages**.
2. Убедитесь, что выбрана ветка `main` и папка `/(root)`.
3. Нажмите Save. Через 2 минуты сайт по ссылке `https://lobikeane220589-cyber.github.io/Dashboard1/` должен заработать.

### Если всё еще не работает:
- Проверьте **регистр букв** в названии репозитория. Если репозиторий называется `dashboard1` (с маленькой), то и в ссылке должна быть маленькая буква.
- Убедитесь, что в файле `vite.config.ts` в строке `base` указано именно имя вашего репозитория.
